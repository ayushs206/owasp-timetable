# Websites Directory

This folder contains all website projects in the OWASP Projects Site workspace.

## Purpose

Use this directory to:

- Host independent website projects.
- Organize work by project folder.
- Keep contributions modular and easy to maintain.

## Current Projects

- `map/`
- `timetable/`

## Adding a New Website Project

1. Create a new folder inside `/websites`.
2. Add a `README.md` in the new project folder.
3. Document the project scope, setup instructions, and contribution rules.

## Contributing to Existing Projects

1. Navigate to the relevant project folder.
2. Follow that project's `README.md` standards.
3. Keep changes focused and project-specific.

## Documentation Standards

- Every project folder under `/websites` should include a `README.md`.
- Update documentation whenever behavior, setup, or structure changes.
- Write clear instructions so other contributors can onboard quickly.
